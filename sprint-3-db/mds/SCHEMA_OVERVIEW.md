# EQUIPO65 — Database Schema Overview

## Summary

| Stat | Value |
|------|-------|
| Total tables | 18 |
| Schema owner | `EQUIPO65` |
| DB engine | Oracle (OCI) |
| Soft deletes | Supported via `IS_DELETED / DELETED_AT` on most tables |
| Identity PKs | All primary keys use `GENERATED BY DEFAULT AS IDENTITY` |

---

## Table Groups

The schema is divided into 6 functional domains:

```
EQUIPO65
├── Identity & Access     → ROLES, TEAMS, USERS, USER_CREDENTIALS
├── Project Management    → PROJECTS, PROJECT_MEMBERS, PROJECT_SPRINTS, SPRINTS
├── Task Tracking         → TASKS, TASK_STATUS_HISTORY, TASK_SPRINT_HISTORY
├── KPIs & Analytics      → KPI_TYPES, KPI_VALUES, LLM_ANALYSIS
├── DevOps                → DEPLOYMENTS, INCIDENTS
└── System                → BOT_INTERACTIONS, AUDIT_LOG
```

## Conventions

- **Soft deletes** — tables with `IS_DELETED / DELETED_AT` never physically delete rows; all queries should filter `NVL(IS_DELETED,'N') = 'N'`.
- **Timestamps** — `CREATED_AT` and `UPDATED_AT` should be set at the application layer on every insert/update.
- **Identity columns** — all PKs use Oracle's `GENERATED BY DEFAULT AS IDENTITY`; do not insert explicit IDs unless seeding reference data.
- **CLOB fields** — `DESCRIPTION`, `RECOMMENDATION`, `MESSAGE`, `RESPONSE` use `CLOB` to support large text payloads from the LLM and bot.
- **Scope pattern** — `KPI_VALUES` and `LLM_ANALYSIS` share a `SCOPE_TYPE` column (`GLOBAL / PROJECT / SPRINT / USER`) with nullable FK columns; only the FK matching the scope will be populated.
